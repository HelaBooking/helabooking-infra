# Environment Setup - Cluster Basic Setup

This document outlines the steps to set up basic components used by other worklods in the cluster. This includes cluster sotrage, monitoring, and security.

## Prerequisites

- A running Kubernetes cluster (version 1.20 or later recommended)
- `kubectl` configured to interact with your cluster

## Step 1: Set Up Rancher (Skipped due to Resource Constraints)

1. Use the provided Terraform module to deploy Rancher in the `cattle-system` namespace.
2. Execute the following command to get the initial admin password:
   ```bash
   kubectl -n cattle-system get secret bootstrap-secret -o go-template='{{.data.bootstrapPassword | base64decode}}'
   ```
3. Access the Rancher UI at `https://rancher.management.ezbooking.lk` and log in using the username `admin` and the password retrieved in the previous step.

## Step 2: Install Longhorn for Cluster Storage

1. Use the provided Terraform module to deploy Longhorn in the `management` namespace.
2. Ensure the persistence volume size and storage class can be seen in the Longhorn UI `https://longhorn.management.ezbooking.lk` using Twingate.
