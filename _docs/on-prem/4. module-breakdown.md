# Module Breakdown

This document provides a detailed breakdown of the various modules in the project, explaining their purpose and functionality.

## Templates

Define reusable templates for common Kubernetes resources (e.g., deployments, services, config maps).

    - cluster-templates
        - deployment
            - deployment.yaml
            - variables.tf
        - service
            - service.yaml
            - variables.tf
        - helm-chart
            - helm-chart.yaml
            - variables.tf
        - pv-claim
            - pv-claim.yaml
            - variables.tf
        - dns-record
            - main.tf
            - secrets.tf
            - dns-record.tf
            - variables.tf

## Environments

Define environment-specific configurations and resources (e.g., namespaces, resource quotas).

    - env-dev
        - deployments.tf: Define kubernetes deployments for the cluster.
        - services.tf: Define kubernetes services for the cluster.
        - networks.tf: Define kubernetes namespaces and network policies.
        - dns-records.tf: Define DNS records for the environment.
        - variables.tf: Define input variables for the environment.
        - pv-claims.tf: Define persistent volume claims for the environment.
